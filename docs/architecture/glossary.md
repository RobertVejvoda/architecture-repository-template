# Architecture Glossary

| Term | Definition | Context | Use When | Avoid |
| --- | --- | --- | --- | --- |
| Architecture Snapshot | Named set of artifact versions reviewed together as a baseline, target, or transition state. | Architecture States | Describing what was accepted together. | Treating a single page version as the whole architecture version. |
| Candidate Architecture | Possible architecture option that is not yet accepted as target or transition architecture. | Architecture States | Comparing options or recording early solution shapes. | Presenting unvalidated ideas as accepted direction. |
| Architecture Change Set | Impact-checked package of architecture updates caused by a material solution change. | Governance | A decision or requirement affects several artifacts. | Using it for spelling, formatting, or local wording edits. |
| Architecture Governance Log | Active queue of board-level architecture questions awaiting routing. | Governance | A question may become a decision, risk, gap, waiver, change set, or delivery action. | Keeping resolved history in the active log. |
| Architecture Contract | Conformance expectations used to verify delivery against accepted architecture. | Phase G | Implementation evidence needs architecture review. | Replacing procurement or legal contracts unless explicitly required. |
| Gap | Difference between baseline/current evidence and the accepted or proposed target. | Architecture States | Something needed by the target is missing or incomplete. | Tracking ordinary backlog tasks without architecture impact. |
| Waiver | Time-boxed accepted exception to a principle, standard, control, or requirement. | Governance | A requirement cannot be met now but is explicitly accepted for a scope and period. | Hiding exceptions in meeting notes. |
