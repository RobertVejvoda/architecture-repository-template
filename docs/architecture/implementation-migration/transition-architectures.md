| Transition | Description | Capabilities Added | Risks |
| --- | --- | --- | --- |
| T1 | Architecture foundation and current-state evidence are organized. | Repository structure, principles, requirements, baseline/target registers. | Incomplete stakeholder review can leave assumptions hidden. |
| T2 | Pilot-ready architecture closes critical operational and security gaps. | Durable state, protected ingress, readiness checks, first operational views. | Accepted limitations must be explicit and time-bound. |
| T3 | Production-ready architecture adds scale, recovery, and governance maturity. | Restore drills, observability, runbooks, compliance evidence, change control. | Increased process burden if governance is not kept lightweight. |
